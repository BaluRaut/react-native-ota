Architecture Decision Record

Secure Self-Hosted OTA Updates for React Native

Executive Summary

We have decided to implement a Self-Hosted, Secure OTA Architecture using hot-updater, AWS S3, CloudFront, and a custom Backend API with Signed URLs.

Verdict: This approach offers the highest level of security for our intellectual property while maintaining the cost benefits of a serverless infrastructure. It eliminates reliance on third-party SaaS providers like CodePush or Expo EAS.

The Challenge

Standard OTA solutions present two main issues for our enterprise use case:

  • Security Risk: Publicly accessible bundle URLs allow anyone to download, reverse-engineer, and inspect our application code.
  • Vendor Lock-in: Services like CodePush or Expo EAS control the infrastructure, limits, and pricing.

The Solution: API + Signed URLs

We are introducing a "Gatekeeper" pattern. The React Native app will no longer talk directly to the public CDN. Instead, it authenticates with our backend, which grants temporary, secure access to the specific update files.
